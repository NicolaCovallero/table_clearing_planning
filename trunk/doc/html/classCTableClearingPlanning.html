Definition at line 402 of file table_clearing_planning.cpp.

void CTableClearingPlanning::computeOnTopPredicates ( double  th1 = 100,
double  th2 = 100,
bool  print = false 
)

To decide if an oject is on top ot the other one, we work with the convex hulls of the projections and the projected points. Since the pow of the kinect make that the on top objects hide part of the bottom objects the strategy is the following: 1) for objet 1 check if its points (choose randomly) lies inside the convex hull of the other one 2) if a point lieas inside the convex hull of the other one, tdo the same with the object 2. 3) if object 1 lies inside the convex hull of the object 2, but object 2 does'nt, the object 1 is on top of object 2. MAYBE A EROSION TECHNIQUE WILL BE REQUIRED IN ORDER TO AVOID SITUATIONS WHERE THE PROJECTED POINTS ARE TOO MUCH CLOSE

Parameters
th1threshold: how many points of the objects which is on the top can lie inside the convex hull of the other
th2The inverse of th1, a correct balance between them we can achieve a nice result

void CTableClearingPlanning::visualComputeBlockPredicates ( Visualizer  viewer,
uint  obj_idx,
uint  dir_idx,
bool  visualization = true,
bool  print = true 
)

This method computes the block predicates for a certain object and a certain direction by detecting collision with the other objects, determining also what object collides with the current one. It also add the viewer all the relative information in order to have a visual feedback of the process of computing the block predicates. The block predicates are then saved in the private member "blocks_predicates".

Parameters
[in]obj_idxIndex of the object
[in]dir_idxIndex of the direction [1,2,3,4]
[in]visualizationTrue to enable the visualization
[in]printTrue to print in the terminal each block predicate
